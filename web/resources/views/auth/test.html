Pages d'authentification UpcycleConnect

ℹ Info: Ces pages sont des fichiers Blade HTML prêts à être intégrés dans Laravel. Cliquez sur les liens ci-dessous pour voir les vues en action.

Inscription

Formulaire d'enregistrement pour les particuliers avec validation complète côté client et gestion des erreurs API.

Validation: Email, password (8+ chars), confirmation
Champs: Nom, Prénom, Email, Téléphone, Ville, Adresse
API: POST /register-particulier
Erreurs: 400, 409 (email existe), 500
Voir le formulaire d'inscription

Connexion

Formulaire de connexion avec gestion des comptes bannit et affichage de la date de déblocage.

Validation: Email, password obligatoires
Champs: Email, Mot de passe
API: POST /login
Erreurs: 401, 403 (banned), 400, 500
Voir le formulaire de connexion

Design System Neo-Brutalism

  • Palette: Cherry Rose (#A4243B), Wheat (#D8C99B), Coffee Bean (#120309), Forest Moss (#244F26), Deep Teal (#18607D), Cream (#F5F0E1)
  • Typographie: Bebas Neue (titres), Outfit (body), DM Mono (labels)
  • Style: Bordures 3px #120309, ombres 5px 5px 0px offset, pas de border-radius
  • Interactions: Hover translate(3px, 3px), focus outline 2px cherry, transitions 0.15s ease
  • Responsive: Mobile-first, breakpoints 600px et 1024px
  • Accessibilité: WCAG AA (3.1:1 contraste), labels liés aux inputs, ARIA labels

✓ Fonctionnalités implémentées

  • Validation live (email format, password min 8, champs obligatoires)
  • Toggle affichage/masquage mot de passe
  • Loading spinner pendant envoi API
  • Gestion complète des réponses API (201, 400, 401, 403, 409, 500)
  • Messages d'erreur contextualisés et détaillés
  • Affichage spécial pour comptes bannit (date déblocage)
  • Stockage JWT en localStorage
  • Redirection automatique post-login/inscription
  • Tous les textes en français avec accents
  • HTML sémantique et accessible
  • JavaScript vanilla (pas de dépendances)
  • Responsive design (mobile, tablet, desktop)

⚠ Configuration nécessaire

  • Ajouter les routes dans routes/web.php:
    Route::get('/register', fn() => view('auth.register'))->name('auth.register');
    Route::get('/login', fn() => view('auth.login'))->name('auth.login');
  • Vérifier que l'API est accessible à http://localhost:8080/api/v1
  • Les URLs API sont codées en dur - à adapter si différentes
  • CORS: Configurer l'API pour accepter les requests depuis le frontend
  • Après login: Créer une route protégée /particulier/dashboard
